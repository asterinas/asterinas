# Issue 05: Global allocator conflict in `cargo-osdk`

## Symptom

```text
error: the `#[global_allocator]` in ostd conflicts with global allocator in: ostd
error: the `#[alloc_error_handler]` in ostd conflicts with allocation error handler in ostd
error: could not compile `aster-kernel-osdk-bin`
```

## Cause

The wrapper crate generated by `cargo-osdk` (`aster-kernel-osdk-bin`) depends on
`ostd` twice: once directly with default features, and once indirectly through
`aster-kernel` with the `cvm_guest` feature. Cargo compiles two copies of `ostd`,
and each copy provides a global allocator, causing a conflict.

## Fix

Build `cargo-osdk` with `OSDK_LOCAL_DEV=1`. This makes the wrapper crate use
local path dependencies for `ostd`, `osdk-frame-allocator`, and
`osdk-heap-allocator`, avoiding the duplicate:

```bash
cd osdk
OSDK_LOCAL_DEV=1 cargo install --path . --locked --force
```

> `OSDK_LOCAL_DEV` must be set when **building** `cargo-osdk`, not when running
> it.

## Verification

After reinstalling `cargo-osdk`, `cargo osdk build` no longer reports the global
allocator conflict.
